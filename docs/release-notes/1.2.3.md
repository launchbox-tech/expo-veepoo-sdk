# 1.2.3

发布日期：待发布

## 变更摘要

- 修复 iOS `Embed VeepooSDK Dynamic Frameworks` 脚本在部分构建环境下因未定义变量导致的失败
- 提升 pod target 构建兼容性，避免 `unbound variable` 中断编译
- 更新使用文档和升级说明

## 修复内容

`1.2.2` 在 podspec 中新增了动态 framework 自动 embed 脚本，使用了：

```sh
set -eu
FRAMEWORKS_DIR="${TARGET_BUILD_DIR}/${FRAMEWORKS_FOLDER_PATH}"
```

在某些 Xcode / CocoaPods 环境下，`FRAMEWORKS_FOLDER_PATH` 可能未注入到该脚本上下文，导致：

```text
line xx: FRAMEWORKS_FOLDER_PATH: unbound variable
Command PhaseScriptExecution failed with a nonzero exit code
```

`1.2.3` 在脚本中增加了环境变量兜底检查，以下变量任一缺失将安全退出：

- `TARGET_BUILD_DIR`
- `FRAMEWORKS_FOLDER_PATH`
- `PODS_TARGET_SRCROOT`

## 升级影响

- 建议所有 iOS 集成方升级到 `1.2.3`
- 不影响 Android
- 不改变 TS API 与返回结构
- iOS Simulator 仍然是“可编译，但不提供真实蓝牙能力”
