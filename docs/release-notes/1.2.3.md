# 1.2.3

**Release status:** Shipped

## Summary

- Fixed the `Embed VeepooSDK Dynamic Frameworks` script failing when build variables were missing
- Improved pod target compatibility (avoid `unbound variable` aborting the build)
- Updated README and upgrade notes

## Details

`1.2.2` added an embed script using:

```sh
set -eu
FRAMEWORKS_DIR="${TARGET_BUILD_DIR}/${FRAMEWORKS_FOLDER_PATH}"
```

In some Xcode / CocoaPods setups `FRAMEWORKS_FOLDER_PATH` is not present in the script environment:

```text
line xx: FRAMEWORKS_FOLDER_PATH: unbound variable
Command PhaseScriptExecution failed with a nonzero exit code
```

`1.2.3` guards required variables and exits safely if any are missing:

- `TARGET_BUILD_DIR`
- `FRAMEWORKS_FOLDER_PATH`
- `PODS_TARGET_SRCROOT`

## Upgrade impact

- All iOS integrators should upgrade to `1.2.3`
- Android unchanged
- TS API and return shapes unchanged
- Simulator remains “builds, no real Bluetooth”
