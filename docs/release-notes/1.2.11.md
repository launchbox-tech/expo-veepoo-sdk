# 1.2.11

## Android breathing-rate realtime test

- **`startBreathingTest` / `stopBreathingTest`** on Android now call the vendor **`startDetectBreath`** / **`stopDetectBreath`** APIs (`IBreathDataListener`, `BreathData`) instead of rejecting with **`CAPABILITY_UNSUPPORTED`**.
- Emits **`breathingTestResult`** with the same payload fields as iOS (`state`, `rawState`, `progress`, `rate`), using **`EDeviceStatus`**-based mapping aligned with existing **`normalizeTestState`** helpers.
- Respects the global realtime-test mutex with kind **`breathing`** (see PRD #66 / issue #67).

## Consumers

- Rebuild the native app after upgrading. Physical Band verification for Android breathing remains **TBD** in the vendor parity matrix until confirmed.
