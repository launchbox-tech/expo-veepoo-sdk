# 1.2.2

发布日期：待发布

## 变更摘要

- 修复 iOS 真机启动时动态 framework 未自动 embed 导致的 `dyld` 崩溃
- 在 package 层区分“只链接”的静态 artifacts 与“必须 embed”的动态 frameworks
- 更新使用文档和升级说明

## 修复内容

`1.2.1` 解决了 `FRAMEWORK_SEARCH_PATHS` 中 `$(inherited)` 的错误引用问题，但 iOS 真机构建仍可能出现：

```text
dyld: Library not loaded: @rpath/JLDialUnit.framework/JLDialUnit
```

原因是 package 只完成了链接，没有确保所需动态 frameworks 被复制到宿主 app 的 `Frameworks` 目录。

`1.2.2` 在 podspec 中补充了集成处理：

- 静态 artifacts：只参与链接，不参与 embed
  - `VeepooBleSDK.framework`
  - `JL_BLEKit.framework`
- 动态 frameworks：在 `iphoneos` 构建下自动复制并签名到 app bundle
  - `ABParTool.framework`
  - `GRDFUSDK.framework`
  - `JLDialUnit.framework`
  - `ZipZap.framework`

## 升级影响

- 建议所有 iOS 集成方升级到 `1.2.2`
- 不影响 Android
- 不改变 TS API 和返回结构
- iOS Simulator 仍然是“可编译，但不提供真实蓝牙能力”

## 发布前检查
