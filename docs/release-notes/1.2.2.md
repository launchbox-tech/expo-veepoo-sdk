# 1.2.2

**Released:** 2026-03-29

## Summary

- Fixed `dyld` crashes on real devices when dynamic frameworks were linked but not embedded
- Separated “link-only” static artifacts from frameworks that must be embedded
- Updated README and upgrade notes

## Details

`1.2.1` fixed `FRAMEWORK_SEARCH_PATHS`, but device builds could still show:

```text
dyld: Library not loaded: @rpath/JLDialUnit.framework/JLDialUnit
```

because dynamic frameworks were not copied into the app bundle.

`1.2.2` updates the podspec:

- Static artifacts (link only, no embed):
  - `VeepooBleSDK.framework`
  - `JL_BLEKit.framework`
- Dynamic frameworks (copied and signed on `iphoneos`):
  - `ABParTool.framework`
  - `GRDFUSDK.framework`
  - `JLDialUnit.framework`
  - `ZipZap.framework`

## Upgrade impact

- All iOS integrators should upgrade to `1.2.2`
- Android unchanged
- TS API and return shapes unchanged
- Simulator remains “builds, no real Bluetooth”

## Pre-release verification

- `pod ipc spec ios/VeepooSDK.podspec`
- `npm run typecheck`
- `npm run build`
