# 1.2.10

## Heart rate alarm (vendor parity)

- **`readHeartRateAlarm()`** — reads current high/low thresholds and enabled flag from the Band (Android `readHeartWarning` / iOS `VPDeviceHeartAlarmModel` read mode).
- **`setHeartRateAlarm(alarm)`** — writes thresholds and on/off (Android `settingHeartWarning` / iOS heart alarm set/close modes). Validates **`HeartRateAlarm`** in JS before BLE (`INVALID_ARGUMENT` for invalid window).
- Both methods emit **`heartRateAlarmData`** after a successful native round-trip with `{ deviceId, data }` (same shape as the `HeartRateAlarm` type).

## Consumers

- Check **`heartRateWarning`** / device capabilities before exposing UI (see `readDeviceFunctions` / `DeviceFunctionPackage1`).
- Run **`npm run typecheck`**, **`npm run build`**, and **`npm test`** (or **`bun test`**) after upgrading.
