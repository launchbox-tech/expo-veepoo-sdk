# 1.2.7

**Released:** Pending

## Summary

- Historical data read progress is a unified integer percent `0–100`
- Progress events fire only when the value changes (no duplicate spam)
- README and version index updated

## Behavior

`readOriginProgress` → `progress.progress` is now:

- An integer from `0` to `100`
- Derived from the legacy `0..1` float by truncation (no rounding)
- Emitted only when the integer changes for a given read on a device
- `100` when the read completes

Normalization happens in TypeScript; Android / iOS native details need not diverge in app code.

## Upgrade impact

- Apps that show historical read progress should adopt `1.2.7`
- If you assumed a `0..1` fraction, switch to integer percent
- Native calling conventions unchanged
