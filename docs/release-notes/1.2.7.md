# 1.2.7

发布日期：待发布

## 变更摘要

- 历史数据读取进度统一改为 `0-100` 的整数百分比
- 进度事件只在数值变化时返回，避免重复派发
- 更新使用文档和版本索引

## 变更内容

`readOriginProgress` 的 `progress.progress` 现在统一为：

- `0-100` 的整数
- 由原始 `0..1` 小数转换而来
- 转换后直接截断，不保留小数部分
- 同一设备同一轮读取过程中，只有当整数值发生变化时才向 App 派发
- 读取完成时固定返回 `100`

这部分统一处理放在 TypeScript 层完成，不需要分别依赖 Android / iOS 原生实现。

## 升级影响

- 建议依赖历史数据读取进度 UI 的业务升级到 `1.2.7`
- 如果旧逻辑按 `0..1` 小数计算进度，需要改为直接使用整数百分比
- 不影响 Android / iOS 原生接口调用方式
