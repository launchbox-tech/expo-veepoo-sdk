# 1.2.0

**Released:** 2026-03-29

## Summary

- Added optional structured logging (disabled by default)
- Apps can consume logs from TypeScript without depending on native log output
- Fixed iOS Simulator link failures caused by device-only vendored frameworks
- Updated root README with logging usage and field list
- Tidied version index

## Logging

`1.2.0` adds lightweight logging hooks useful for:

- Scan start/stop
- Device connect/disconnect
- Password verification
- Data reads (start, progress, completion)
- Manual tests (start/stop/results)
- Listener errors

## Usage

```ts
import VeepooSDK, { type LogEntry } from '@gaozh1024/expo-veepoo-sdk';

VeepooSDK
  .setLogEnabled(true)
  .setLogger((entry: LogEntry) => {
    console.log('[veepoo-log]', entry.level, entry.scope, entry.action, entry);
  });
```

New methods:

- `setLogEnabled(enabled: boolean): VeepooSDK`
- `isLogEnabled(): boolean`
- `setLogger(logger: ((entry: LogEntry) => void) | null): VeepooSDK`

Fields:

- `timestamp`
- `level`
- `scope`
- `action`
- `platform`
- `message`
- `deviceId`
- `data`
- `error`

## Upgrade impact

- Default behavior unchanged (logging off)
- No change to `VeepooSDK` method return shapes
- No change to native calling conventions
- Simulator builds compile; real Bluetooth still requires a device
- Enable logging during app init if you need debug traces
