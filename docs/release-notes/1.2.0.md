# 1.2.0

发布日期：2026-03-29

## 变更摘要

- 补充统一日志能力，默认关闭，按需开启
- App 可通过 TS 层接收结构化日志，不需要直接依赖原生日志输出
- 修复 iOS Simulator 构建时因 device-only vendored frameworks 导致的链接失败
- 更新根目录使用文档，补充日志使用方式和字段说明
- 同步整理版本文档索引

## 新增日志能力

`1.2.0` 新增了一层轻量日志接口，适合排查以下问题：

- 扫描开始和停止
- 设备连接和断开
- 密码校验结果
- 数据读取开始、进度和结果
- 手动测试开始、停止和结果事件
- 事件监听器抛错

## 使用方式

```ts
import VeepooSDK, { type LogEntry } from '@gaozh1024/expo-veepoo-sdk';

VeepooSDK
  .setLogEnabled(true)
  .setLogger((entry: LogEntry) => {
    console.log('[veepoo-log]', entry.level, entry.scope, entry.action, entry);
  });
```

新增方法：

- `setLogEnabled(enabled: boolean): VeepooSDK`
- `isLogEnabled(): boolean`
- `setLogger(logger: ((entry: LogEntry) => void) | null): VeepooSDK`

日志统一字段：

- `timestamp`
- `level`
- `scope`
- `action`
- `platform`
- `message`
- `deviceId`
- `data`
- `error`

## 升级影响

- 默认行为不变，日志默认关闭
- 不影响现有 `VeepooSDK` 方法返回结构
- 不影响原生层调用方式
- iOS Simulator 现在可以完成工程编译，但不提供真实蓝牙能力
- 如果业务需要接入调试日志，可直接在 App 初始化时开启
