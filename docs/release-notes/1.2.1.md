# 1.2.1

发布日期：2026-03-29

## 变更摘要

- 修复 iOS app target 的 `FRAMEWORK_SEARCH_PATHS` 配置错误
- 避免 `$(inherited)` 被错误引用后破坏 React / React Native 预编译 framework 的搜索路径
- 更新使用文档和版本说明

## 修复内容

`1.2.0` 为解决 iOS Simulator 链接问题，在 podspec 中按 SDK 条件注入了 `FRAMEWORK_SEARCH_PATHS`。

但 `1.2.0` 的配置把 `$(inherited)` 写成了带引号的字符串：

```ruby
"$(inherited)"
```

这会让 Xcode 把继承来的 search paths 当成单个路径值处理，可能导致以下问题：

- `search path '...React-Core-prebuilt...' not found`
- `framework 'React' not found`

`1.2.1` 已修复为未引用的 inherited 形式：

```ruby
$(inherited)
```

## 升级影响

- 建议所有 iOS 集成方升级到 `1.2.1`
- 不影响 Android
- 不改变 TS API 和返回结构
- iOS Simulator 仍然是“可编译，但不提供真实蓝牙能力”
