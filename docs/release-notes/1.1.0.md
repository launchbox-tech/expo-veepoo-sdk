# 1.1.0

发布日期：待发布

## 变更摘要

- 整理并重写对外使用文档
- 将 Android / iOS 常用数据返回统一收敛到 TypeScript 层处理
- 保留原生层更贴近各自 SDK 文档的返回语义
- 补充蓝牙状态、连接状态和测试结果的统一事件处理

## 升级影响

`1.1.0` 的重点是“对 App 暴露的返回结构统一化”。如果你的业务代码一直通过：

```ts
import VeepooSDK from '@gaozh1024/expo-veepoo-sdk';
```

来调用 SDK，那么升级后通常不需要改调用方式，但需要注意以下几点。

## 需要关注的变化

### 1. 方法返回优先使用 TS 统一结构

以下方法的返回现在会先经过 TypeScript 层归一化：

- `requestPermissions`
- `verifyPassword`
- `readBattery`
- `readDeviceFunctions`
- `readSocialMsgData`
- `readDeviceVersion`
- `readSleepData`
- `readSportStepData`
- `readOriginData`
- `readDaySummaryData`
- `readAutoMeasureSetting`
- `modifyAutoMeasureSetting`

建议不要直接依赖 native 侧字段细节，也不要绕过 `VeepooSDK` 去使用内部 native module。

### 2. 事件 payload 统一

以下事件 payload 现在会在 TS 层统一：

- `bluetoothStateChanged`
- `readOriginProgress`
- `deviceFunction`
- `deviceVersion`
- `passwordData`
- `socialMsgData`
- `originFiveMinuteData`
- `originHalfHourData`
- `sleepData`
- `sportStepData`
- `heartRateTestResult`
- `bloodPressureTestResult`
- `bloodOxygenTestResult`
- `temperatureTestResult`
- `stressData`
- `bloodGlucoseData`
- `batteryData`

### 3. 测试状态建议只认统一后的 `state`

不同平台原生 SDK 的状态枚举并不完全一致。升级后，建议业务代码优先使用统一后的：

- `result.state`
- `data.state`

而不是依赖平台原始状态值。

### 4. 连接监听建议

升级后，连接相关推荐同时监听：

- `deviceDisconnected`
- `connectionStatusChanged`
- `bluetoothStateChanged`

这样可以同时覆盖设备断开和系统蓝牙关闭。

## 不兼容风险

如果你的代码直接依赖以下行为，升级前请自查：

- 直接访问 native module
- 依赖 Android / iOS 各自的原始字段名
- 依赖某个平台专有的测试状态值
