# 1.1.0

**Released:** 2026-03-29

## Summary

- Reworked public-facing documentation
- Converged common Android / iOS data returns in TypeScript while leaving native layers closer to vendor semantics
- Clarified Bluetooth state, connection state, and unified test result handling

## Upgrade impact

`1.1.0` focuses on **unified shapes exposed to the app**. If you already import:

```ts
import VeepooSDK from '@gaozh1024/expo-veepoo-sdk';
```

you usually keep the same call sites; note the items below.

## What changed

### 1. Prefer TS-normalized return values

These methods now pass through TypeScript normalization:

- `requestPermissions`
- `verifyPassword`
- `readBattery`
- `readDeviceFunctions`
- `readSocialMsgData`
- `readDeviceVersion`
- `readSleepData`
- `readSportStepData`
- `readOriginData`
- `readDaySummaryData`
- `readAutoMeasureSetting`
- `modifyAutoMeasureSetting`

Avoid relying on raw native fields or bypassing `VeepooSDK`.

### 2. Unified event payloads

These events are normalized in TS:

- `bluetoothStateChanged`
- `readOriginProgress`
- `deviceFunction`
- `deviceVersion`
- `passwordData`
- `socialMsgData`
- `originFiveMinuteData`
- `originHalfHourData`
- `sleepData`
- `sportStepData`
- `heartRateTestResult`
- `bloodPressureTestResult`
- `bloodOxygenTestResult`
- `temperatureTestResult`
- `stressData`
- `bloodGlucoseData`
- `batteryData`

### 3. Test status: prefer unified `state`

Native enums differ per platform. Prefer:

- `result.state`
- `data.state`

instead of raw platform-specific codes.

### 4. Connection listeners

Listen together for:

- `deviceDisconnected`
- `connectionStatusChanged`
- `bluetoothStateChanged`

to cover both Band disconnect and system Bluetooth off.

## Breaking-risk checklist

Review before upgrading if you relied on:

- Direct native module access
- Raw Android / iOS field names
- Platform-specific test status codes only found on one OS

## Pre-release verification

For this release:

- `npm run typecheck`
- `npm test -- --runInBand`
- `npm run build`
- `npm pack --dry-run`
